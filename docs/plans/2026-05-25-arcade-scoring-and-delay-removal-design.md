# 系統設計文件：移除延時與街機化寬鬆評分優化

本設計文件記錄「跳舞動作相似度評分軟體」之移除延時比對、放寬關節角度比對上限至 $\pi$（極大化容忍差），以及調整 Sigmoid 曲線中位數與斜率以杜絕低分與個位數分值出現之技術規格。

## 1. 移除 0.3 秒反應延遲 (Remove Reaction Delay)
依照使用者最新需求，我們移除先前新增的 0.3 秒延遲比對，改回使用影片當下的真實時間戳記：
$$\text{currentTime} = \text{referenceVideo.currentTime}$$
這將使畫面計算直接對齊影片。

## 2. 街機化寬鬆評分設計 (Arcade Scoring Calibration)

### 2.1 角度相似度改用 $\pi$ 作為除數 (Maximum Angle Tolerance)
* 為了完全避免玩家在擺放動作時，因為部分關節的合理位移被重罰為 0 分，我們將除數放寬至最大物理極限 $\pi$（約 3.14，即 $180^\circ$ 錯置）：
  $$\text{Similarity} = 1 - \frac{\text{diff}}{\pi}$$
  * 當關節夾角偏離大達直角 ($90^\circ$) 時，相似度仍可保留高達 $0.5$ (50%)。
  * 這也是 MediaPipe Pose 角度比對最常規、最溫和的比對算式，能保證各關節的相似度分數不會輕易斷崖式下跌。

### 2.2 Sigmoid 曲線參數調整 (Preventing Single-digit Scores)
原先中位數偏高且斜率較陡，導致當平均相似度略高於 `minSimilarity` 時（如 50%），計算出的分數只有個位數 (1~9 分)。
* **調整方案**：
  * **計分底線**：將 `minSimilarity` 調降為 **0.40**。
  * **中位數 (midpoint)**：將中位數 $m$ 下調為 **0.70** (原 0.78)。當玩家的平均關節匹配度為 70% 時即可拿到 50 分。
  * **斜率 (k)**：將斜率 $k$ 降為 **12.0** (原 15.0)。使評分曲線的上升更加溫和、分布更均勻。
* **效果**：
  * 當玩家的平均相似度在 55% 附近時，將可拿到 $15 \sim 25$ 的正常基本分數，完全告別卡在 $1 \sim 9$ 個位數的尷尬情形。
  * 同時，若玩家**完全站立不動**去比對大擺幅舞蹈，平均相似度仍會輕易跌破 40%，維持防呆機制。
