# 系統設計文件：評分精準度校準與效能深度優化

本設計文件記錄「跳舞動作相似度評分軟體」之評分判定防呆、評語門檻重劃、以及相機影格尺寸縮小等效能深度優化的技術規格與實作方案。

## 1. 評語等級門檻重劃
依照使用者最新需求，將實時跳出的 4 個評語等級門檻修改為：
* **Perfect**：$\ge 80$ 分
* **Great**：$70 \sim 79$ 分 (即 $\ge 70$ 且 $< 80$)
* **Good**：$55 \sim 69$ 分 (即 $\ge 55$ 且 $< 70$)
* **Miss**：$< 55$ 分

---

## 2. 解決「站立不動分數仍過高」之防呆校準設計

### 2.1 原因分析
* **無效角度稀釋**：原先比對採用了 24 組角度，其中包含多個活動度極低或靜態的角度（如頭肩協調角、下軀幹扭角、腋下跨度、領口等）。當玩家站立不動，即使手腳動作完全沒對上，這些大軀幹/靜態部位的相似度仍然接近 $1.0$，稀釋了四肢錯誤動作的扣分效果。
* **角度誤差容忍度過寬**：原先的相似度公式為 $1 - (diff / \pi)$。即使夾角相差高達 90 度 ($\pi/2$)，該關節相似度仍算作 $0.5$（50分），導致極度嚴重的動作錯誤仍能獲得過高分數。

### 2.2 解決方案
1. **精簡至 12 組核心動態關節**：
   只比對高活動度的手肘、肩膀、膝蓋、臀部、手腕、腳踝等 12 組決定跳舞主要姿勢的核心角度，排除其餘 12 組微小或靜態的角度干擾。
2. **重設角度誤差函數 (嚴格扣分機制)**：
   我們將夾角差異除以可容忍的最大角度差 $\theta_{max}$，並限制相似度最低為 0。
   我們設定最大角度容許值 $\theta_{max} = 1.6$ 弧度（約 $90^\circ$）。
   $$sim = \max\left(0, 1 - \frac{diff}{1.6}\right)$$
   * 如此一來，只要單個關節角度差距大於 $90^\circ$，其相似度即判定為 $0$ 分。
   * 即使玩家站立不動，面對正在大範圍活動的舞蹈參考動作，其四肢角度的巨大落差將直接觸發 $0$ 分，使整體相似度大幅下降，順利消除「站立不動拿高分」的異常問題。

## 3. 相機視訊卡頓深度優化 (相機影格縮圖下採樣)

### 3.1 原因分析
* MediaPipe Pose 對輸入圖片尺寸非常敏感。當直接呼叫 `poseModel.send({ image: webcamVideo })` 時，若攝影機的原始解析度較高 (如 1080p 或 720p)，MediaPipe 將花費大量 CPU/GPU 資源在主線程中進行影格縮放、轉換與處理，造成主線程瞬間阻塞 (Blocking)，使得 60fps 畫布渲染的 `requestAnimationFrame` 被迫推遲，引發鏡頭畫面與影片卡頓。

### 3.2 解決方案
* **離屏畫布預先縮小 (Pre-downscaling)**：
  在記憶體中建立一個隱藏的 `smallCanvas` (尺寸固定為 $256 \times 256$ 像素)。
  在每次進行 AI 推理前，先在 CPU/GPU 負擔極低的 Canvas 2D 環境下將相機影格快速繪製並縮小到 `smallCanvas`：
  ```javascript
  smallCtx.drawImage(webcamVideo, 0, 0, 256, 256);
  ```
  然後將 `smallCanvas` 傳遞給 `poseModel.send` 進行推理。
* **優勢**：
  * MediaPipe 的內建 Pose 網路輸入層解析度即為 $256 \times 256$。預先餵入剛好大小的 Canvas 可以省去 MediaPipe 內部極耗資源的縮放處理，讓 AI 推理速度提升 2~3 倍，主線程完全不卡死，畫面達到極致順暢。
  * 由於 MediaPipe Pose landmarks 回傳的是 $0.0 \sim 1.0$ 的百分比座標，輸入尺寸的縮小完全不會影響偵測的座標精準度，因此與原先 $640 \times 480$ 畫布的繪圖與計算 100% 相容。
