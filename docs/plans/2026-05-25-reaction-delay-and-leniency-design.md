# 系統設計文件：反應延時補償與寬鬆角度比對

本設計文件記錄「跳舞動作相似度評分軟體」之玩家反應時間補償（0.3 秒延遲比對）與角度容忍度寬鬆化調整（2.2 弧度最大閾值）之技術規格與實作方案。

## 1. 0.3 秒玩家反應時間延時比對 (Reaction Lag Compensation)

### 1.1 設計動機
* 玩家在看著參考影片跳舞時，存在約 0.3 秒的視覺反應與模仿延遲。如果系統進行「完全實時同步時間軸」的比對，玩家會因為反應慢 0.3 秒而被判定為動作落後不符，導致分數偏低。
* **改進方案**：在擷取參考影片特徵時，將時間軸回溯 0.3 秒：
  $$\text{Delayed Time} = \max(0, \text{currentTime} - 0.3)$$
  系統會將玩家「當下」的動作，與參考影片「0.3 秒前」的姿勢進行比對，從而完美補償人類的自然生理反應時間。

### 1.2 程式碼調整
在 `handleLiveWebcamResults` 中，查詢 `poseFeatures` 的時間戳記改為 `currentTime - 0.3`。

## 2. 角度容忍度寬鬆化調整 (Lenient Joint Similarity)

### 2.1 設計動機
* 先前的防呆校準將角度最大容許差設為 1.6 弧度 (約 $91^\circ$)，這在肢體擺放有小幅度偏差時會被過度懲罰。例如：舞蹈影片中有些動作是「身體直立但雙手揮舞」，若玩家站姿稍微偏了一點（如雙腳站得比影片寬，或髖關節角度有些許不同），這 12 個關節中的下半身關節會直接拿到近 0 分，導致整體平均分數暴跌。
* **改進方案**：將最大夾角容許誤差值 $\theta_{max}$ 調寬至 **2.2 弧度** (約 $126^\circ$)：
  $$\text{Similarity} = \max\left(0, 1 - \frac{\text{diff}}{2.2}\right)$$
  * 當單個關節偏離在 $30^\circ$ 以內時，相似度依然維持在 $0.75 \sim 1.0$ 的高分區間。
  * 偏離到 $90^\circ$ 時仍保留 $0.28$ (28%) 的相似度，直到偏離超過 $126^\circ$ 才會判定為 $0$。
  * 如此一來，即使玩家站姿有小幅誤差，依然能藉由手部精準動作拿到不錯的分數，有效避免「身體不動手有動被判 0 分」的過嚴現象，且保留了「完全站立不動時仍會被判定為低分 (Miss)」的防呆特性。
