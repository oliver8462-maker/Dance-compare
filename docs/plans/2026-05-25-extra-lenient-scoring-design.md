# 系統設計文件：進一步放寬角度比對與最低評分限制調整

本設計文件記錄「跳舞動作相似度評分軟體」之比對誤差除數調寬（防範個別關節過嚴扣分）與最低相似度基準線調降（杜絕異常出現 0 分）之技術規格與實作方案。

## 1. 角度比對除數放寬至 2.6 弧度 (Relaxed Angle Mismatch Divisor)

### 1.1 問題分析
先前將單個核心關節的最大容忍角度差設為 2.2 弧度 (約 $126^\circ$)。在實際測試中，有些影片即便身體不動只動雙手，玩家的身體重心站姿、雙腿稍微站開一些（可能偏離了幾十度），就會導致下半身關節直接被判定為 $0$ 分。如果同時有兩三個關節拿 $0$ 分，平均相似度就會被大幅拉低。

### 1.2 解決方案
* 我們將單個關節最大容許的角度落差調寬至 **2.6 弧度** (約 $149^\circ$)。
  $$\text{Similarity} = \max\left(0, 1 - \frac{\text{diff}}{2.6}\right)$$
  * 在這個設定下，即使玩家的站姿腳步與影片有 $60^\circ$ (1.05 弧度) 的偏移，該關節相似度仍可保留高達 $0.6$ (60%)。
  * 即使某一關節呈現直角錯置 ($90^\circ \approx 1.57$ 弧度)，該關節依然能拿到 $0.4$ (40%) 的相似度，直到擺放角度偏離接近半圓 ($149^\circ$) 以上才會獲得 $0$ 分。
  * 這將大幅度放寬對於局部站姿或微小偏移的判定，讓玩家做對手部動作時仍然能夠獲取高分。

## 2. 最低相似度門檻調降至 0.48 (Preventing Harsh Zero Scores)

### 2.1 問題分析
在 `math-utils.js` 的 `scaleScore` 函數中，存在防呆門檻限制：
```javascript
export function scaleScore(avgSim, minSimilarity = 0.55) {
  if (avgSim < minSimilarity) return 0;
  ...
```
原先預設的最低相似度基準線 `minSimilarity` 為 `0.55`。只要玩家這 12 個關節的平均相似度低於 55%，該影格的分數就會直接被無情歸零（直接得 0 分）。在肢體動作豐富或速度快的舞蹈中，玩家很容易因為慢了一點或某些姿勢偏差，導致平均相似度落入 50% 左右，從而頻繁看到「0 分」跳出，體驗極差。

### 2.2 解決方案
* 將 `minSimilarity` 預設值下調至 **0.48**。
* **效果**：這拉低了計分的底線。只要玩家的平均姿勢符合度達到 48% 以上（即便動作非常不精準），依然能開始獲得分數（如 5~15 分），徹底消除遊玩過程中突兀出現的「0 分」，提供玩家更強烈的正向鼓勵。
* 同時，若玩家**完全站立不動**去比對大擺幅舞蹈，平均相似度仍會輕易跌破 0.48，因此仍然會合理判為 0 分 (Miss)，保留了防呆機制。
